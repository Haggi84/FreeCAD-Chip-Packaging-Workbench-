🛠 Troubleshooting

GDS Import Issues

No layers appear after import

Cause: The LYP file layer IDs do not match the GDS layer IDs, or all layers are invisible in the LYP.
Fix: Open the LYP in KLayout and check that at least one layer is set to visible. Make sure the LYP was exported from the same technology as the GDS.

Wrong colors / all grey

Cause: Match KLayout is off, and no IHP map was loaded, so the material heuristic assigns generic grey.
Fix: Enable Match KLayout in the import options, or load a .map file to enable material-based styling.

Import is extremely slow

Cause: The GDS contains millions of small polygons (fill / dummy metal).
Fix: Enable Contacts-only 3-D to collapse non-pad layers into a single bounding box solid. Alternatively increase the minimum area filter in the layer selector.

Auto PIN contact points are misplaced

Cause: PIN markers (DT=2) in the GDS are at pad corners rather than pad centers, or no MAP file was loaded.
Fix: Load a .map file so the tool uses the IHP layer name strategy (Strategy 1). Contact points are placed at the bounding-box center of the DT=0 pad polygon — verify the GDS has pad-sized DT=0 shapes on the top metal layer.

Wire Bonding Issues

"Wire Bond" button is greyed out

Cause: Missing prerequisites — the tool needs both die-side contact points AND housing/leadframe contact points.
Fix:
  • Die-side: orange ContactPoint_* markers must exist (from GDS import or Define Contact Points)
  • Housing-side: yellow contact_point_housing_* markers must exist (from Set Contact Points on Face)

Wire geometry is hollow / incorrect

Cause: The swept-pipe end caps failed to close (can happen with very small wire radii or near-degenerate paths).
Fix: Increase the wire diameter in the bonding configurator. The code falls back to a shell (open pipe) if end-cap creation fails — check the FreeCAD Python console for a "Wire end-cap failed" warning.

Contact points not moving with the chip

Cause: Move/Rotate Chip is in GDS Chip Objects mode but some contact points have unexpected names.
Fix: Switch to All Document Objects mode in the Chip Transform dialog, or use Current Selection (Ctrl+A to select all first).

Toolbar / UI Issues

Advanced toolbar does not appear when toggled

Cause: FreeCAD may have saved the toolbar as permanently hidden in its user settings.
Fix: Go to View → Toolbars and make sure "GDSII Advanced Tools" is checked.

Icons are missing (blank buttons)

Cause: SVG/PNG icon files are missing from resources/icons/.
Fix: Re-install the workbench. Check the FreeCAD Python console for "Icon not found" warnings on startup.

Help guide shows "Content Not Found"

Cause: HTML files are missing from resources/html/.
Fix: Re-install the workbench or copy the HTML files from the repository into that folder.

Session Issues

Session replay fails with an error

Cause: The GDS / LYP / MAP / STEP file paths stored in the session file no longer exist (moved or deleted files).
Fix: Open the .dipas file in a text editor, find the path fields, and update them to the current file locations.

"No Session to Save" message

Cause: No design actions have been recorded yet in this FreeCAD session.
Fix: Perform at least one action (Load GDSII, Leadframe, etc.) before saving. Note: if you loaded a session from a file and made no new changes, there is also nothing new to save.

Getting Further Help

Python Console: All workbench operations print status messages and errors to the FreeCAD Python console (View → Panels → Python console). Check there first when something does not work as expected.
Report an issue: Visit the project repository and open an issue with:
  • FreeCAD version
  • Operating system
  • Steps to reproduce
  • Python console output
