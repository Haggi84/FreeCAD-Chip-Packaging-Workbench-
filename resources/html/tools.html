🔧 Tool Reference

Main Toolbar

Load GDSII

Purpose: Import a GDS layout file with color, technology, and 3-D options.

Input files: .gds (required), .lyp (required), .map (optional), stackup .xml (optional)

Import options:
  • Match KLayout — use exact LYP fill/frame colors
  • Highlight bondable — gold color for PIN/LEFPIN/PAD layers
  • Extrude to 3-D — polygon solids with PDK-accurate Z-heights
  • Contacts-only 3-D — full geometry for bond pads, single body box for rest
  • Auto PIN contacts — automatic contact-point markers on detected bond pads
Created objects: Layer_{name}_{id}, IC_Body_Solid, GDS_Pin_Instances, GDS_PINs_{layer}, ContactPoint_{n}

Move / Rotate Chip

Purpose: Translate and rotate all GDS chip objects as a group.

Scope: GDS Chip Objects / All Document Objects / Current Selection

Controls: Arrow buttons for XY, ▲▼ for Z; ±X/Y/Z rotation buttons around bounding-box center

Keyboard shortcuts (when dialog is focused):
←/→ = ±X    ↑/↓ = ±Y    PgUp/PgDn = ±Z
Shift + ←/→ = ±Rz    Shift + ↑/↓ = ±Rx    Shift + PgUp/PgDn = ±Ry

Restore Original — undoes all moves made in the current dialog session

Set Contact Points on Face

Purpose: Place contact point markers interactively on selected pad faces.

Phase 1: Ctrl+click faces in the 3-D view → click Generate Grid
Phase 2: Click grid points to select them → click OK

Result: contact_point_housing_{n} markers (yellow) placed at selected grid positions.
These housing-side contact points activate the Wire Bond command automatically.

Contact Point Browser

Purpose: Dock panel listing all contact points grouped by type.
Groups: Die-side (orange), Housing (yellow)
Interaction: Click a row to highlight the corresponding marker in the 3-D view.

Wire Bond

Purpose: Create 3-D solid bond wires between die pads and package leads.

Prerequisites: At least one die-side contact point (orange) and one housing contact point (yellow).

Workflow: Configure wire parameters → click die contact point → click housing contact point → wire is created

Wire geometry: Swept solid pipe with end caps — true solid, not a hollow shell.
Created objects: BondWire_{n}

Wire Bump Configurator

Purpose: Place parametric bump shapes at bond wire endpoints.

Shapes:
ShapeParametersTypical Use
Ball BondBall radius, neck radius, heightStandard thermosonic bonding
Wedge BondWidth, length, heightUltrasonic Al bonding
Stitch BondRadius, heightSecond bond in ball-wedge sequence
Nail HeadTop radius, base radius, heightHeavy wire bonds
Netlist browser: Shows all BondWire_* objects with net name, start/end CP, and wire length.
Check rows and click Place Bumps to place at both endpoints.
Created objects: WireBump_{shape}_{n}

Session ▾

Save Design Session: Writes all recorded actions to a .dipas JSON file.
Load Design Session: Reads a .dipas file and replays all actions to reconstruct the design.

Recorded actions: GDS import, Leadframe config, Layer-on-Leadframe, Housing, Wire bonding config, Center Leadframe

Advanced Tools ⇄

Purpose: Toggle the visibility of the Advanced Tools toolbar.
The advanced toolbar is hidden by default to keep the main toolbar uncluttered.

Advanced Toolbar

Leadframe Configurator

Purpose: Generate a parametric semiconductor package leadframe.

Package types: QFN (Quad Flat No-lead), QFP (Quad Flat Package), BGA (Ball Grid Array)

Parameters: Frame size (L×W×T), lead count per side, lead pitch, lead width/length, die-attach paddle size, material

Leadframe Library (STEP Import)

Purpose: Import a real STEP package model from the IHP mirror-semi online library.

Workflow: Browse library → select package → confirm orientation face → confirm die-attach face → STEP model is placed at the correct position in the GDS document

Housing Configurator

Purpose: Generate a transparent IC package housing around the leadframe.

Parameters: Wall thickness, clearance, housing height, lid (yes/no), transparency, alignment posts, material

Layer on Leadframe

Purpose: Import GDS layers and stack them directly onto an existing leadframe.

Features: Auto-scale to fit die-attach paddle, rotation, mirror, Z-offset, combined GDS + leadframe in one document

Define Contact Points

Purpose: Batch-place contact point markers at the top-face centre of selected GDS layer objects.

Usage: Select one or more Layer_* objects in the 3-D view, then click this button.
Each selected object gets one ContactPoint_{n} marker at its highest Z face centre.
